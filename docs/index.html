The High-Performance Engine.

Reproq Worker is a multi-threaded execution engine written in Go. It handles the heavy lifting of polling, heartbeat management, and process execution.

Low Latency

Atomic polling with SKIP LOCKED ensures sub-millisecond task pickup.

Self-Healing

Built-in lease reaper and heartbeat system prevents zombie tasks.

Installation

Reproq provides pre-built binaries for Linux, macOS, and Windows. You can download them manually or use the Django helper.

Manual Install (Go)

go build -o reproq ./cmd/reproq

Cross-Platform Binaries

Download from the Releases page. Binaries follow the pattern reproq-{os}-{arch}.

Command Reference

worker

Starts the execution node.

./reproq worker --dsn $DB_URL --concurrency 20 --metrics-port 9090

beat

Starts the cron scheduler.

./reproq beat --dsn $DB_URL --interval 30s

replay

Re-enqueues a task.

./reproq replay --id 12345

Prometheus Monitoring

Enable the built-in metrics exporter to monitor throughput and error rates.

# Start with metrics enabled
./reproq worker --metrics-port 9090

Exposed metrics include:

  • reproq_tasks_processed_total: Counter of tasks by queue and status.
  • reproq_task_duration_seconds: Histogram of execution times.
  • Standard Go runtime metrics (goroutines, heap, etc).

Deployment

For production deployment, it is recommended to use systemd to manage the worker and beat processes. Reproq provides an automated command to generate these service files for you:

python manage.py reproq systemd

See the Deployment Guide for manual templates and best practices.

Note: Reproq Worker is designed to work in tandem with Reproq Django, which provides the task definition and monitoring interface.
